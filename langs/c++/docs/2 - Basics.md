## Tokens

# Keyword 
- Reserved words with predefined meanings
- if, for, include, return, etc

# Identifiers
- User-defined names for variables, functions, and arrays
- Sum, Result, addNumbers

# Literals
- Fixed values that do not change during executions
- 3.142, 'A', 0

# String
- Sequence of characters enclosed in " "
- "PROGRAM", "FALSE"

# Operators
- Symbols that perform operaions on variables and values
- + & <

# Punctuators
- Character with special meanings in c
- {} ; . etc

## Additionals

# General Rules
- Use meaningfull names
- Start with letter or underscore
- Avoid using reserved keywords
- Use lowercase for variable names

# Variable Naming
- Use lowercase with underscore for variable names
- Use prefixs for more clarity
- ex : total_sum, student_age 

# Functional Naming
- Use verbs describe actions
- Prefer camelCase
- ex : calculateArea(), printResult()

# Constant Naming
- Use uppercase with underscores
- ex : MAX_VALUE, PI_VALUE

# Macro Naming( #define )
- Use uppercase for macros
-  ex : #define MAX_LENGTH 100, #define MIN_VALUE 0

# Structure & Enum Naming
- Use PascalCase for types
- ex : enum Color {
    RED, GREEN, BLUE
    };
- ex : typedef struct StudentData;
